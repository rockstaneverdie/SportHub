## 2. Backend: Django Framework

### 2.1 Описание

Django — высокоуровневый веб-фреймворк на Python, основанный на архитектурной парадигме Model-Template-View (MTV), которая является вариацией классической Model-View-Controller (MVC).

### 2.2 Обоснование выбора

1. **Полнофункциональность**
   - Встроенная объектно-реляционная модель (ORM) для работы с БД
   - Встроенная система аутентификации и авторизации
   - Встроенный администраторский интерфейс (Django Admin)

2. **Безопасность**
   - Защита от CSRF-атак (Cross-Site Request Forgery)
   - Защита от SQL-инъекций благодаря параметризованным запросам ORM
   - Встроенное хеширование паролей

3. **Производительность на этапе разработки**
   - Быстрый цикл разработки
   - Автоматическая генерация миграций БД
   - Горячая перезагрузка при изменении кода

4. **Экосистема**
   - Большое сообщество разработчиков
   - Богатаяликата расширений и пакетов
   - Обширная документация и учебные материалы

### 2.3 Основные компоненты Django, используемые в проекте

#### 2.3.1 Models (Модели)

Реализованы следующие модели:

- **Category** — категория товаров с иконкой для визуального идентификации
- **Product** — товар с полной информацией: название, описание, цена, старая цена (для скидок), изображение, наличие на складе
- **Cart** — корзина покупок, привязанная к пользователю (один-к-одному)
- **CartItem** — элемент корзины, связывающий товар и количество
- **Order** — заказ с набором статусов
- **OrderItem** — элемент заказа с информацией о цене на момент заказа

#### 2.3.2 Views (Представления)

Реализованы функциональные представления (Function-Based Views, FBV):

- `home()` — главная страница с рекомендациями
- `catalog()` — каталог товаров с фильтрацией и поиском
- `product_detail()` — детальная информация о товаре
- `cart_view()` — просмотр корзины
- `add_to_cart()`, `remove_from_cart()`, `update_cart()` — операции с корзиной
- `checkout()` — оформление заказа
- `my_orders()`, `order_detail()` — управление заказами
- `login_view()`, `register_view()`, `logout_view()` — аутентификация
- `account_view()` — управление профилем

#### 2.3.3 Templates (Шаблоны)

Используется система шаблонизации Django с поддержкой:
- Наследования шаблонов (`{% extends %}`)
- Блоков динамического контента (`{% block %}`)
- Условного отображения (`{% if %}`)
- Циклов (`{% for %}`)
- Контекстных процессоров для глобальных переменных

#### 2.3.4 Admin Interface

Используется встроенный Django Admin для управления данными:
- Просмотр и редактирование товаров
- Управление заказами и их статусами
- Управление пользователями

## 3. Frontend: HTML5, CSS3, JavaScript

### 3.1 HTML5

Используется для создания семантической структуры документов с современными элементами:
- Семантические теги: `<nav>`, `<section>`, `<footer>`
- Метаданные для адаптивности: `<meta name="viewport">`
- Встроенные иконки через Unicode: 🏃 (спорт), 🛒 (корзина)

### 3.2 CSS3

Организован в едином файле `style.css` с использованием:

1. **CSS Custom Properties (переменные)**
   ```css
   :root {
     --orange: #FF4500;      /* Основной акцентный цвет */
     --black: #000000;       /* Фоновый цвет */
     --dark2: #1A1A1A;       /* Вторичный фоновый цвет */
     --gray: #666666;        /* Текст вторичной информации */
     --white: #FFFFFF;       /* Основной цвет текста */
   }
   ```
   Использование переменных обеспечивает консистентность дизайна и облегчает его изменение.

2. **Flexbox и CSS Grid**
   - Flexbox для выравнивания элементов в навигации
   - CSS Grid для расположения товаров в каталоге (автоматическое заполнение с минимальной шириной)

3. **Адаптивный дизайн (Responsive Design)**
   - Media-запросы для различных устройств
   - Мобильная версия (макет в один столбец)
   - Десктопная версия (多-столбцовый макет)

4. **Анимации и переходы**
   - `transition: var(--transition)` для плавных эффектов
   - `@keyframes slideIn` для анимации появления уведомлений
   - `transform: translateY()` для эффектов при наведении

### 3.3 JavaScript

Минимальное использование JavaScript для:
- Автоматического скрытия уведомлений: `setTimeout(() => mc.style.display = 'none', 4000)`
- Подтверждения действий: `onsubmit="return confirm(...)"`

**Обоснование:** предпочтение отдано серверному рендерингу, что упрощает разработку и безопасность.

## 4. База данных

### 4.1 SQLite (Разработка)

Используется для локальной разработки благодаря:
- Отсутствию необходимости установки отдельного сервера
- Встроенности в Python
- Достаточной производительности для разработки

### 4.2 PostgreSQL (Production)

Рекомендуется для production благодаря:
- **Надёжности** — ACID-гарантии
- **Масштабируемости** — поддержка больших объёмов данных
- **Функциональности** — поддержка полнотекстового поиска, индексирования, предотвращения race conditions через `SELECT FOR UPDATE`
- **Производительности** — встроенная оптимизация запросов

### 4.3 Схема данных

```
User (Django built-in)
├── Cart (OneToOne)
│   └── CartItem (ForeignKey)
│       └── Product (ForeignKey)
│           └── Category (ForeignKey)
└── Order (ForeignKey)
    └── OrderItem (ForeignKey)
        └── Product (ForeignKey)
```

## 5. Pillow: Обработка изображений

### 5.1 Назначение

Pillow (Python Imaging Library) используется для:
- Загрузки изображений товаров через `ImageField` в Django
- Сжатия и оптимизации изображений автоматически
- Валидации формата файлов

### 5.2 Обоснование выбора

- Стандартная библиотека для работы с изображениями в Python
- Встроенная интеграция с Django ORM
- Поддержка всех основных форматов (JPG, PNG, GIF, WebP)

## 6. Дополнительные технологии

### 6.1 Python (язык программирования)

- **Версия:** 3.13+ (или 3.12 как альтернатива, избегая 3.14 из-за несовместимости)
- **Обоснование:** 
  - Читаемый синтаксис ускоряет разработку
  - Мощная экосистема пакетов
  - Поддержка современных парадигм программирования

### 6.2 Django ORM (Object-Relational Mapping)

Используется вместо raw SQL для:
- **Безопасности** — параметризованные запросы предотвращают SQL-инъекции
- **Переносимости** — код работает с SQLite, PostgreSQL, MySQL без изменений
- **Оптимизации** — поддержка `select_related()`, `prefetch_related()` для избегания N+1 проблемы
- **Удобства** — декларативное определение схемы вместо SQL

### 6.3 Django Migrations

Система управления версиями схемы БД:
- Автоматическое отслеживание изменений в моделях
- Возможность отката и применения миграций
- Версионирование изменений БД

## 7. Стек кэширования

Реализована многоуровневая система кэширования:

### 7.1 LocMemCache (разработка)

```python
CACHES = {
    'default': {
        'BACKEND': 'django.core.cache.backends.locmem.LocMemCache',
        'LOCATION': 'unique-snowflake',
    }
}
```

### 7.2 Redis (production)

```python
CACHES = {
    'default': {
        'BACKEND': 'django_redis.cache.RedisCache',
        'LOCATION': 'redis://127.0.0.1:6379/1',
    }
}
```

**Обоснование:** Redis обеспечивает быстрое кэширование между запросами и поддерживает распределённые системы.

## 8. Веб-сервер

### 8.1 Development

Встроенный Django development server для разработки.

### 8.2 Production

**Gunicorn** — WSGI HTTP Server:
- Синхронная обработка запросов
- Многопроцессная архитектура (количество workers = CPU * 2 + 1)

**Nginx** — reverse proxy и статический файловый сервер:
- Раздача статических файлов с кэшированием
- SSL/TLS терминация
- Балансировка нагрузки
